# 参考资料与设计来源

Phi 和这门课程并不是在真空中设计出来的。在确定 Model、Harness、Session、Context、Tool、
Event、Hook 和 Multi-agent 等边界时，我们比较了多个 Agent Harness、coding agent、开放协议和
框架 API。

下面记录的不只是一份书目，还包括：

- 每个项目或协议具体启发了 Phi 的哪一部分；
- Phi 最终采用了什么；
- Phi 又主动拒绝或改变了什么。

出现在这里表示我们认真参考过这份资料，不代表 Phi 接受它的全部架构，也不代表实现代码来自该项目。
真正决定 Phi 当前行为的，仍然是 Phi 自己的代码、测试和设计文档。
